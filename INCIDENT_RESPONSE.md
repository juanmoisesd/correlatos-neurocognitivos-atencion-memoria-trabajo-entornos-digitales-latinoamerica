# Incident Response Plan

## Detection
Incidents may be detected via CI/CD failures, security scans, or community reports.

## Response Steps
1. **Identification**: Confirm the breach or vulnerability.
2. **Containment**: Limit the impact (e.g., deprecate the affected version).
3. **Eradication**: Fix the underlying issue.
4. **Recovery**: Release a patched version with updated checksums/signatures.
5. **Lessons Learned**: Update threat models and policies.
